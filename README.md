# 说明文档：AlecaFrameWebhookForwarder
## 基于Python的Alecaframe消息提醒信息转发工具（Webhook）

### 一、所需环境

1. **Python**
   - 下载：`python-3.13.3-amd64.exe`
2. **AlecaFrame的安装包**
   - （或者官网下载：https://alecaframe.com/ ）
   - 下载：`AlecaFrame - Installer.exe`

---

### 二、使用说明

#### 1. 程序完整介绍
- **程序名称**：`AlecaFrameWebhookForwarder.py`
- **版本**：v1.0.1
- **开发者**：Roc
- **技术栈**：Python 3.x + Flask + Requests
- **适用场景**：Warframe玩家通过企业微信（或任何有Webhook接口的工具）接收AlecaFrame获取到的Waframe游戏通知

**【核心功能】**
① **消息代理转发**：
- 实时接收AlecaFrame产生的游戏通知
- 通过企业微信群机器人Webhook转发格式化消息
- 支持Markdown富文本呈现（含时间戳标注）

② **智能过滤机制**：
- 内置关键词黑名单（默认过滤"Warframe.x64.exe"等进程信息）
- 重复消息拦截（2分钟内相同内容自动过滤）
- 异常符号清洗（自动修复*****异常星号格式）

③ **安全防护**：
- 消息缓存上限控制（最大1000条防内存溢出）
- 自动缓存清理（过期消息定时清除）
- 线程安全锁保障（防止多线程冲突）

④ **可定制化配置**：
- 支持自定义Webhook地址
- 可扩展关键词过滤列表
- 消息缓存时长调节（默认120秒）
- 端口灵活配置（默认9090）。
> （如果你需要修改一个端口，可以在Py主程序里直接把所有的“9090”字符，全部替换为你需要的端口号）

**【技术特点】**
- ◇ 采用轻量级Flask框架构建Web服务
- ◇ OrderedDict实现LRU缓存淘汰机制
- ◇ 正则表达式深度清洗消息内容
- ◇ 多线程安全锁保障数据一致性
- ◇ 完善的错误日志记录（含堆栈跟踪）
- ◇ HTTP状态码规范响应（200/500等）

**【版本信息】**
- **当前版本**：v1.0.1
- **更新日期**：2026年4月22日
- **依赖库要求**：
  - Flask >= 2.0.0
  - Requests >= 2.26.0

#### 2. 环境准备

① **Python安装步骤（必须）**：
1. 进入项目目录下的"可能用到的Python安装包"文件夹。
2. 运行安装包：双击运行 `python-3.x.x-amd64.exe`（64位系统专用）。
3. 安装时注意事项：
   - **必须勾选 "Add Python 3.x to PATH" 选项**
   - 建议选择 "Install Now" 默认安装路径（蓝色按钮）
4. 验证安装（可选）：
   - 同时按键盘 `Win+R` 键，输入 `cmd` 按回车。
   - 在黑色窗口输入（可复制粘贴）：
     `python --version`
   - 再输入：
     `pip --version`
   - 两个命令都显示版本号才算成功，否则重启电脑后重试。

② **安装依赖库（必须）**：
1. 右键点击屏幕左下角Windows图标，选择"Windows PowerShell(管理员)"。
2. 在蓝色窗口输入以下命令（逐行执行）：
   cd "你的路径"
   pip install flask requests
3. 看到 `Successfully installed` 之类的提示表示成功。

如果对于输入上述的文件路径有困难，可以尝试直接使用以下防呆方式：
> 1. 打开你保存本代码的位置（文件名：`AlecaFrameWebhookForwarder.py`）。
> 2. 打开存放这个代码的文件夹，在顶部的文件路径地址栏里清空内容，直接输入“`cmd`”并回车，就会弹出一个已经定位好路径的黑窗口。
> 3. 输入 `pip install flask requests` 并回车。
> 4. 等待安装依赖完毕即可。

③ **企业微信设置（必须）**
（可以不是企业微信的机器人，是任何有Webhook功能的机器人均可）
- 登录企业微信电脑客户端。
- 进入目标群聊 -> 点击群聊窗口右上角的三个点 -> 点击“消息推送”。
- 创建新机器人，记录生成的 **Webhook URL**。

#### 3. Python主程序配置（必须）

> **编辑提醒：**
> - 建议右键点击该文件，选择**‘打开方式’ -> ‘记事本’**。
> - 修改完成后，直接按 `Ctrl+S` 保存即可。
> - **千万不要选择‘另存为’**，也不要在代码前面随意增删空格，Python语言对代码前面的空格比较敏感。

1. 用文本编辑器打开 `AlecaFrameWebhookForwarder.py`
2. 修改以下配置项：
   - **第28行**（必须按你自己的地址修改）：将 `WECHAT_WEBHOOK_URL` 替换为你的机器人Webhook地址。
   - **第29行**（可不修改）：根据需要修改 `EXCLUDE_KEYWORDS` 排除关键词集合。
   - **第32行**（可不修改）：`CACHE_EXPIRY` 可调整消息缓存时间（默认120秒）。

---

#### 4. 运行程序（标准方式）

① **启动服务（在程序目录打开CMD执行）**：
> **如何在本Python程序所在的目录打开CMD？**
> 1. 打开你保存本代码的位置（文件名：`AlecaFrameWebhookForwarder.py`）。
> 2. 打开存放这个代码的文件夹，在顶部的文件路径地址栏里清空内容，直接输入“`cmd`”并回车，就会弹出一个已经定位好路径的黑窗口。

打开CMD后，输入命令并回车：
`python AlecaFrameWebhookForwarder.py`

② **保持命令行窗口处于开启状态**。
此后，程序将监听 `http://127.0.0.1:9090/wechat_forward`。

---

#### 5. AlecaFrame配置（必须）

① 打开AlecaFrame设置 -> 找到 **Webhook** 选项卡。
② 在AlecaFrame的设置中，设置Webhook地址为：`http://127.0.0.1:9090/wechat_forward`。
③ 确保AlecaFrame的提醒、Webhook功能已启用。

---

#### 6. 验证测试（可选）

① 发送测试消息（在本程序所在文件夹的路径中，新开CMD执行）：
`curl -X POST http://127.0.0.1:9090/wechat_forward -d "content=test_message_from_cmd"`

② 发送了测试消息后，检查企业微信群是否收到格式化的消息。

### 三、常见问题

**1. 无法接收消息时检查：**
- 确保Windows防火墙允许你指定的端口号（默认是9090）的通信。
- 检查企业微信机器人Webhook地址是否正确。
- 确认AlecaFrame的Webhook功能已启用。

**2. 消息重复过滤机制：**
- 2分钟内相同内容不会重复发送。
- 缓存最多保留1000条消息。
- 可通过修改 `CACHE_EXPIRY` 的值，调整过滤时长。

### 四、注意事项

1. **Python安装特别提醒**：
   - 安装前可以关闭360等杀毒软件（有人反映会出问题，但我没有出问题）。
   - **找不到python命令的解决方法**：
     - a. 卸载Python后重新安装。
     - b. 必须勾选 `Add Python to PATH` 选项。
     - c. 安装时务必使用默认安装路径。
   - 使用管理员身份运行CMD执行pip命令（有人反映不用管理员模式运行会出问题，但我没有出问题）。

2. 保持程序持续运行期间不要关闭命令行窗口。

3. **如需修改监听端口**（默认9090），需同步修改：
   - 代码里的port参数。
     > （如果你需要修改一个端口，可以在Py主程序里直接把所有的“9090”字符，全部替换为你需要的端口号）
   - AlecaFrame设置里的Webhook的地址端口号（与上面的端口号同步修改）。
